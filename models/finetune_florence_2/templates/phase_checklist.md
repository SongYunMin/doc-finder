# Florence-2 Fine-tuning 체크리스트

## 1단계. 작업 정의

- [ ] 출력이 자유문장이 아니라 표준 태그 문자열인지 확정했다.
- [ ] prompt prefix를 정했다.
- [ ] 정답 예시를 최소 10개 작성했다.

## 2단계. taxonomy

- [ ] 핵심 태그를 30개 이상 정의했다.
- [ ] 금지 표기와 허용 표기를 구분했다.
- [ ] 팀 내에서 같은 이미지를 봤을 때 태그 합의가 가능한지 확인했다.

## 3단계. 데이터셋 구축

- [ ] gold label을 사람이 직접 검수했다.
- [ ] 모델 출력이나 기존 후처리 결과를 정답으로 재사용하지 않았다.
- [ ] notes 필드에 애매한 사례를 기록했다.

## 4단계. 데이터 분할

- [ ] train/validation/test를 나눴다.
- [ ] test셋은 이후 수정 대상에서 제외했다.
- [ ] 유사 이미지 누수 가능성을 검토했다.

## 5단계. baseline

- [ ] zero-shot Florence-2 결과를 저장했다.
- [ ] raw output과 후처리 결과를 같이 봤다.
- [ ] 어떤 유형에서 실패하는지 패턴을 정리했다.

## 6단계. 실행 준비

- [ ] processor 입력 포맷을 검증했다.
- [ ] GPU 메모리 한계를 확인했다.
- [ ] 사용할 체크포인트를 확정했다.

## 7단계. smoke run

- [ ] 작은 subset으로 1차 학습을 돌렸다.
- [ ] loss가 정상적으로 내려가는지 봤다.
- [ ] validation 출력이 완전히 무너지지 않는지 확인했다.

## 8단계. 본 학습

- [ ] baseline과 동일한 test셋으로 평가했다.
- [ ] exact match와 tag F1을 비교했다.
- [ ] 검색 품질 개선 여부까지 확인했다.
